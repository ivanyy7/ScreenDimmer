# ScreenDimmer — план

## Цель

Небольшое приложение для Windows: полное затемнение всех мониторов сплошным чёрным (оверлеи), переключение горячей клавишей, иконка в трее, один экземпляр процесса, без прав администратора.

## Маршрут (MVP)

1. Проект WPF на .NET 8, user-mode, манифест `asInvoker`.
2. Низкоуровневый хук клавиатуры (`WH_KEYBOARD_LL`) для выбранной клавиши (сейчас F8).
3. Окна-оверлеи на каждый монитор (`Screen.AllScreens`), позиционирование через Win32 при необходимости.
4. `NotifyIcon`: переключение и выход.
5. Single instance через именованный `Mutex`.

## Изменения в плане

- Реализованы по желанию: **иконка** приложения и трея (`Assets/app.ico`, совместимость с GDI+ / `System.Drawing.Icon` — см. `tools/rebuild-app-ico.ps1`), **автозапуск** через ярлык в папке «Автозагрузка» пользователя.

## Этап 2 — главное информационное окно (после стабильного MVP)

**Статус:** **реализовано** (подэтапы 2.1–2.7); первоначальное планирование и договорённости — март 2026.

**Цель этапа:** дать пользователю понятное окно «о программе» и настройку темы оформления, не меняя ядро затемнения и не усложняя фоновый режим работы.

### Принципы реализации (уровень senior)

Реализация этапа 2 и каждого подэтапа **2.1–2.7** ведётся в духе **опытного разработчика**: не «красивый прототип», а **предсказуемое, лёгкое по коду, рабочее** приложение.

- **Надёжность:** явные сценарии (трей, потоки, закрытие окна vs выход процесса), без скрытых побочных эффектов; границы ответственности модулей — читаемые.
- **Простота:** только тот код и зависимости, которые нужны для требований; без оверинжиниринга и «на будущее», если это не в плане.
- **Проверяемость:** после каждого подэтапа — сборка без регрессий и **критерий готовности из таблицы** выполнен или явно зафиксированы оставшиеся риски.
- **Сохранение инвариантов:** не ломать ядро ScreenDimmer — `DimmerService`, хук F8, single instance, выход только через Exit в трее, фоновая работа без лишних окон.
- **Сопровождение:** комментарии **на русском** в местах, где неочевидно (например интеграция WinForms `NotifyIcon` с WPF и `Dispatcher`).

Этот раздел задаёт **как** делать; подэтапы ниже — **что и в каком порядке**.

### Функциональные требования

1. **Главное окно (WPF `Window`):** размер порядка **480×320** px (ориентир — небольшое окно, как у типичного установщика). Содержимое:
  - заголовок: **ScreenDimmer**;
  - краткое описание назначения (затемнение всех мониторов);
  - инструкция по **горячей клавише F8** (вкл/выкл затемнение);
  - переключатель **тёмная / светлая тема** интерфейса окна; **по умолчанию — тёмная**.
2. **Запуск:** при старте приложения окно **не показывается**; приложение ведёт себя как сейчас — только трей и фоновая логика (хук, затемнение).
3. **Трей:** **левый клик (ЛКМ)** по иконке в трее — показать окно; повторный ЛКМ при открытом окне — **скрыть** (toggle видимости), без лишних режимов.
4. **Панель задач:** **никогда не показывать** кнопку приложения в панели задач (`ShowInTaskbar = false` и при показе, и при скрытии окна). Единственная точка доступа к UI из оболочки — **иконка в трее**.
5. **Кнопки окна:** **нет кнопки закрытия (крестика)**. Есть только **одна кнопка сворачивания** («минус» / minimize): по нажатию окно **полностью скрывается** (не остаётся на рабочем столе и **не** появляется в панели задач). Завершение процесса — **только** из контекстного меню трея → **Exit** (как сейчас).
6. **Закрытие окна:** событие закрытия окна **не должно** завершать приложение: при необходимости — `Closing` с отменой и скрытием (`e.Cancel = true` + `Hide()`), чтобы случайный жест не завершил утилиту.

### Сохранение настроек (сразу в объёме этапа)

- **Тема** (тёмная по умолчанию, светлая по выбору) — **сохранять между сеансами** (например `Properties.Settings` user-scope или JSON/XML в профиле пользователя).
- **Положение и размер окна** — **сохранять и восстанавливать**, чтобы окно не «прыгало» при каждом открытии (в разумных пределах: не выходить за пределы экрана при смене конфигурации мониторов).

### UI / UX

- **Без анимаций** и без декоративных эффектов: статичный, **читаемый** интерфейс, нормальные отступы, **достаточный контраст** в обеих темах, заметный переключатель темы.
- Приоритет — **функциональность и ясность**, не «красота ради красоты».

### Технические ориентиры (надёжность)

- Интеграция с существующим `NotifyIconService.cs` (WinForms `NotifyIcon`): обработка **ЛКМ** на UI-потоке WPF через `Dispatcher`, по аналогии с уже существующими событиями трея.
- Ядро: `App.xaml.cs` (`ShutdownMode.OnExplicitShutdown`), `DimmerService.cs`, хук — **не ломать**; новое окно не должно менять семантику выхода и single instance.
- Окно без «лишней» кнопки закрытия: **кастомная шапка** или `WindowChrome` / шаблон с одной кнопкой minimize — выбрать простой устойчивый вариант без лишней сложности.

### Критерии готовности этапа

- Окно открывается и закрывается по ЛКМ с трея, сворачивается кнопкой minimize без появления в панели задач.
- Тема и (при реализации) геометрия окна переживают перезапуск приложения.
- Выход приложения — только через **Exit** в меню трея; фоновая работа (F8, затемнение) сохраняется.
