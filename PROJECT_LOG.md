# PROJECT_LOG — ScreenDimmer

## Текущий статус

- **Работает (по задумке):** WPF-приложение .NET 8 с затемнением всех мониторов чёрными оверлеями, переключение по **F8** (низкоуровневый хук), трей с пунктами Toggle и Exit, один экземпляр через `Mutex`, без админских прав.
- **В фокусе:** убедиться, что на ПК установлен **.NET 8 SDK** и успешно выполняется `dotnet build`; при необходимости — донастройка под конкретную клавиатуру (различение Fn+клавиша от обычной F8).

## Ключевые шаги и история

- День 1, 12:00 — Создан MVP: структура WPF, `LowLevelKeyboardHook`, `DimmerService`, `OverlayWindow`, `NotifyIconService`, single instance, манифест asInvoker.
- День 1, 12:05 — Горячая клавиша переведена с F6 на **F8** (`vkCode == 0x77`), тексты в трее обновлены («Toggle Dimmer (F8)»).
- День 1, 12:10 — Добавлены `PLAN.md`, `PROJECT_MAP.md`, `PROJECT_LOG.md` для фиксации состояния и маршрута.

## Технические решения и грабли

- **RegisterHotKey** не подходит для комбинаций с **Fn** — используется **WH_KEYBOARD_LL** и отслеживание нужного `vkCode` на KEYUP, чтобы не ловить автоповтор.
- На части ноутбуков ОС не различает «голую» F8 и Fn+F8 — поведение зависит от OEM; при необходимости можно ужесточить фильтр по `flags` в `KBDLLHOOKSTRUCT`.
- Сборка через `dotnet` на машине без установленного SDK завершается ошибкой «No .NET SDKs were found» — лечится установкой SDK 8.x (например 8.0.419), перезапуск терминала после установки обычно достаточен; полная перезагрузка ПК не обязательна.
- Установщик SDK предупреждает о **Visual Studio 2022 ≥ 17.8.3** при работе с .NET 8 *внутри VS* — на использование SDK из Cursor/CLI это не накладывает обязательных ограничений.
