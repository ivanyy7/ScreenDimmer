# PROJECT_LOG — ScreenDimmer

## Текущий статус

- **Стабильный MVP:** затемнение всех мониторов по **F8**, трей (Toggle, автозапуск, Exit), **иконка** корректно отображается у **exe** и в **трее** (`Assets/app.ico` + встроенный ресурс, формат совместим с GDI+), single instance (`Mutex`), без админских прав.
- **Сборка:** .NET 8 SDK; перед `dotnet build` при необходимости закрывать запущенный exe (блокировка файла). **Release** и ручной запуск проверены; затемнение и снятие затемнения работают; клики по чёрному слою не проходят вниз (ожидаемо).

## Технические решения и грабли

- **RegisterHotKey** не подходит для комбинаций с **Fn** — используется **WH_KEYBOARD_LL** и отслеживание нужного `vkCode` на KEYUP, чтобы не ловить автоповтор.
- На части ноутбуков ОС не различает «голую» F8 и Fn+F8 — поведение зависит от OEM; при необходимости можно ужесточить фильтр по `flags` в `KBDLLHOOKSTRUCT`.
- Сборка через `dotnet` на машине без установленного SDK завершается ошибкой «No .NET SDKs were found» — лечится установкой SDK 8.x (например 8.0.419), перезапуск терминала после установки обычно достаточен; полная перезагрузка ПК не обязательна.
- Установщик SDK предупреждает о **Visual Studio 2022 ≥ 17.8.3** при работе с .NET 8 *внутри VS* — на использование SDK из Cursor/CLI это не накладывает обязательных ограничений.
- Автозапуск через ярлык в `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup` — не требует прав администратора; при запущенном exe **повторная сборка Release** может падать из‑за блокировки файла (закрыть приложение перед `dotnet build`).
- Иконка: один `app.ico` в проекте для exe и трея. Старый ICO из конверта PNG давал `ArgumentException` у `new Icon()` (GDI+ не принимал формат) — трей падал на системную иконку; исправлено пересборкой через `tools/rebuild-app-ico.ps1`. Для произвольного PNG — `tools/png-to-ico.ps1 -InputPng ...` (проверять загрузку `Icon` вручную).
